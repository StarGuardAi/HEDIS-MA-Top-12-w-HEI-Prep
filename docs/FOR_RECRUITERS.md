## 🎬 Demo Talking Points

### Context Engineering
> "The system uses 3-layer hierarchical context engineering. Layer 1 caches domain knowledge for 1 hour—HEDIS specs, clinical guidelines. Layer 2 caches measure-specific data for 5 minutes. Layer 3 uses fresh query data. This achieves 82% cache hit rate, reducing query time by 60% and costs by 61%."

### Agentic RAG
> "For complex queries like 'Calculate ROI for diabetes interventions across all measures,' the system decomposes it into steps: retrieve domain knowledge, query member data, calculate ROI, validate results. It self-corrects failed validations with 98% success rate, ensuring accurate, compliant responses."

### Security
> "All processing happens on-premises. No PHI ever leaves the system. We use local LLMs (Ollama) and on-premises vector search (ChromaDB). Complete audit trails log every query. This eliminates the need for Business Associate Agreements—the system is BAA-free."
